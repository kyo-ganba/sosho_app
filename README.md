# 送迎表自動生成ツール — プロジェクト引継ぎドキュメント

## プロジェクト概要
放課後等デイサービス・児童発達支援事業所（4館：Ⅰ〜Ⅲ・Ⅴ番館、各70〜100名、計300名規模）向けの送迎表作成自動化ツール。StreamlitでWebアプリとして動作し、Streamlit Community Cloudで無料公開中。

**GitHub**: `github.com/kyo-ganba/sosho_app`
**本番URL**: Streamlit Community Cloud（kyo-ganbaアカウント）

---

## ファイル構成

```
sosho_app/
├── app.py            # メインアプリ（Streamlit画面定義・全4ページ）
├── master.py         # 利用者マスタ管理（CRUD・変更履歴・リタリコCSV取込）
├── routing.py        # 送迎ルート自動生成ロジック
├── excel_export.py   # Excel出力（タイムライン縦型・2シート構成）
├── color_config.py   # カラー設定管理
├── requirements.txt  # 依存ライブラリ
└── data/             # 実行時データ保存先（自動生成）
    ├── {館}_master.csv       # 利用者マスタ
    ├── {館}_staff.json       # スタッフ情報
    ├── {館}_vehicles.json    # 車両情報
    ├── {館}_colors.json      # カラー設定
    └── history/{館}/         # マスタ変更履歴（最新20件）
```

## アプリの画面構成（4ページ）

### 1. 当日入力・送迎表生成
- 対象日・長期休みトグルを選択
- 利用曜日マスタから参加者を自動チェック
- 当日の欠席・追加・迎え時刻変更を入力
- スタッフ勤務状況を確認
- 「送迎ルートを自動生成」ボタン
- ルート確認・手動修正
- Excelダウンロード（ファイル名: `YYYYMMDD_{館}送迎表.xlsx`）

### 2. 利用者マスタ管理
- リタリコCSV取込（文字コード自動判定：UTF-8/Shift-JIS/CP932）
- 手動編集（data_editorで追加・削除・編集）
- 変更履歴（保存ごとに自動バックアップ、最新20件保持、復元可能）

### 3. 車両・スタッフ設定
- 車両名・定員を登録
- スタッフ名・運転可否・勤務時間を登録

### 4. カラー設定
- Excel出力の各セル色をカラーピッカーで変更
- 館ごとに設定を保存
- デフォルトリセット機能

## データ構造

### 利用者マスタ（MASTER_COLUMNS）
```
氏名, フリガナ, 区分（放デイ/児発）, 地区,
迎え先（平日）, 迎え先（長期休み）,
迎え時刻（平日）, 迎え時刻（長期休み）,
送り先, 送り時刻,
利用曜日（例: 月水金）, 住所, 特記事項, 備考
```

### スタッフ（JSON）
```json
[{"氏名": "酒井", "運転可": true, "勤務時間": "9:00-18:00", "備考": ""}]
```

### 車両（JSON）
```json
[{"車両名": "セレナ", "定員": 7, "備考": ""}]
```

## Excelフォーマット仕様

### レイアウト
- **方式**: タイムライン縦型（行＝時刻、列＝車両）
- **シート構成**: 午前シート（〜12:59）＋ 午後・送りシート（13:00〜）
- **印刷設定**: A4横・横1ページ固定・余白0.5cm

### 列構成（車両あたり4列）
```
迎/送バッジ | 氏名 | 送迎先 | 運転者
```

### 特殊行
- **チャイルドシート行**: 迎え便に児発利用者が含まれる場合、その便の直下に「チャイルドシート◯脚」を表示
- **特記事項行**: 特記事項登録済みの利用者行の直下に黄色帯で展開
- **はしご便矢印行**: 同一車両で迎え便が連続する場合に「↓ はしご」行を自動挿入

### 色識別
- 迎え（放デイ）: バッジ赤系
- 迎え（児発）: バッジ青系
- 送り: バッジ緑系
- 送迎先: 白背景（色なし）
- 運転者: 白背景（色なし）
- 特記事項: 黄色背景
- 急遽追加: オレンジ系・点線枠

## ルーティングロジック（routing.py）

1. 参加者を「迎え」「送り」に分け、時刻順にソート
2. 同じ地区の利用者を同じ車両にまとめる（greedy割当）
3. 定員-1（ドライバー分）を超えたら次の車両へ
4. 長期休みフラグで迎え先・迎え時刻を切り替え

**改善予定**: 「A校のあとB校に間に合うか」の時間×距離チェック（現状は地区ベースのみ）

## 開発の経緯メモ
- リタリコ（支援記録SaaS）から利用者情報CSVをエクスポートして取込む運用
- 元のExcelフォーマット（車両×時刻の複合レイアウト）から、タイムライン縦型に変更
- 車両とドライバーは固定ではなく便ごとに入れ替わる
- 1台あたり7便以上動くため、A4横1枚に5台を全部収めるのは現実的でない
  → 午前/午後の2シート構成に落ち着いた
- チャイルドシートは児発（児童発達支援）利用者のみ必要
